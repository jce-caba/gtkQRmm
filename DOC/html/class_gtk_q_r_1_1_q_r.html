QrMatrix GtkQR::QR::get_qrmatrix ( )

Get the character matrix representing the QR.

Returns
character matrix

A QR image is made up of a series of rectangles called modules, each value of the matrix is a coordinate (X,Y) that represents the different modules, being value '1' for foreground color and '0' for background color.

void GtkQR::QR::set_dynamic_margin ( bool  set_dynamic)

sets whether the margin is always the same ,default:true

Parameters
set_dynamicfalse is is always the same , true otherwise

The QR standard establishes that at least 4 image margin modules are necessary ,but the number of modules varies from 21x21 in version 1 to 170x170 in version 40 .if the parameter is set to false the margin is calculated in the worst case 21x 21 , if it is set to true the margin will depend on the number of modules or qr version

◆ set_UTF8()

void GtkQR::QR::set_UTF8 ( bool  set_UTF8)

Allows the encoding of characters in UTF8 ,default:true.

Parameters
set_UTF8:true to allow UTF8, false otherwise

In Extended Channel Interpretation allows the encoding of characters in UTF8 when using characters not included in ISO/IEC 8859-1, if you disable this option then it will use the following encoding : ISO/IEC 8859-2, ISO/IEC 8859-3, ISO/IEC 8859-4, ISO/IEC 8859-5, ISO/IEC 8859-6, ISO/IEC 8859-7, ISO/IEC 8859-8, ISO/IEC 8859-9, ISO/IEC 8859-10, ISO/IEC 8859-11, ISO/IEC 8859-13, ISO/IEC 8859-14, ISO/IEC 8859-15, ISO/IEC 8859-16 or JIS X 0201
.With UTF8 you will have all the characters but occupying more space for the encoding, in case of using ISO/IEC 8859-2 to JIS X 0201 the encoding uses less space but not all the characters are possible
